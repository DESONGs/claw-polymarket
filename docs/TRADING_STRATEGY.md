# 地缘政经驱动的多资产预测框架：三工具协同方法论

## 一、核心理念与能力边界

### 范式定义

本框架的核心假设是：**地缘政治与政经事件是资产价格波动的第一驱动力，而预测市场盘口是对这些事件概率最高效的群体定价机制。**

传统金融分析依赖滞后的经济数据和主观研判。本框架引入预测市场作为**实时概率锚**——它浓缩了大量参与者对特定事件发生概率的真金白银下注，比任何民调、专家意见或情绪指标都更具信息密度。

我们不在预测市场上交易。我们**读取**预测市场，将其作为分析全球资产价格走势的独特信号源。

### 三工具在新范式下的角色

```
┌───────────────────────────────────────────────────────────┐
│                  OpenClaw（事件感知层）                      │
│  职责：地缘政经事件的检测、分类与结构化                        │
│  输入：Twitter 实时流 / 新闻聚合 / 官方声明                  │
│  输出：结构化事件描述 + 事件类型标签 + 紧急度评分              │
└───────────────────────────────────────────────────────────┘
                           │
                           ▼
┌───────────────────────────────────────────────────────────┐
│            Polymarket Skill（概率信号层）                    │
│  职责：提取群体对事件发生概率的共识定价                        │
│  输入：事件关键词 → 关联预测市场盘口                          │
│  输出：事件概率（midpoint）/ 概率趋势（price_history）/       │
│        市场确信度（spread + book depth）                     │
│  三重角色：群体智慧 · 领先指标 · 情绪温度计                   │
└───────────────────────────────────────────────────────────┘
                           │
                           ▼
┌───────────────────────────────────────────────────────────┐
│              Claude AI（综合分析与预测层）                    │
│  职责：融合事件信息与概率信号，推导资产价格影响                 │
│  输入：结构化事件 + 预测市场信号 + 资产影响映射规则            │
│  输出：方向概率判断 / 多情景分析 / 综合信号仪表盘              │
└───────────────────────────────────────────────────────────┘
```

**关键区分**：
- OpenClaw 回答"**发生了什么？**"
- Polymarket Skill 回答"**市场认为这件事有多大概率发生/持续/升级？**"
- Claude AI 回答"**这对 BTC / 标普 / 原油 / 美元意味着什么？**"

### 覆盖资产类别

| 资产类别 | 典型标的 | 主要敏感事件类型 |
|----------|----------|-----------------|
| 加密货币 | BTC, ETH | 监管政策、央行利率、流动性环境、ETF 审批 |
| 美股与指数 | SPX, NQ, 个股 | 央行政策、贸易关税、财政政策、地缘冲突 |
| 大宗商品 | WTI 原油, 黄金, 铜 | 地缘冲突、制裁、OPEC 决策、供应链中断 |
| 外汇 | DXY, EUR/USD, USD/CNY | 央行政策分化、贸易谈判、资本流动 |

资产不是固定标的，而是**由事件类型动态决定**——检测到地缘冲突事件时聚焦原油和黄金，检测到 FOMC 信号时聚焦股指和加密。

## 二、三工具协同架构：预测分析管线

### 完整数据流

```
                    ┌──────────────────────────────┐
                    │       外部触发源                │
                    │  ┌─────────┐  ┌────────────┐  │
                    │  │定时扫描  │  │事件推送触发  │  │
                    │  └────┬────┘  └─────┬──────┘  │
                    └───────┼─────────────┼─────────┘
                            │             │
                            ▼             ▼
              ┌─────────────────────────────────────┐
  Stage 1     │     事件感知与结构化（OpenClaw）       │
              │  ① 信息源扫描（Twitter / News）        │
              │  ② 事件识别与去噪                      │
              │  ③ 事件分类打标（类型 + 紧急度）        │
              └──────────────────┬──────────────────┘
                                 │ 结构化事件

              ┌─────────────────────────────────────┐
  Stage 2     │  事件-资产映射（Claude AI 初筛）       │
              │  ① 匹配影响链规则                      │
              │  ② 确定受影响资产类别                   │
              │  ③ 生成预测市场搜索关键词               │
              └──────────────────┬──────────────────┘
                                 │ 关键词 + 受影响资产列表
                                 ▼
              ┌─────────────────────────────────────┐
  Stage 3     │ 预测市场信号提取（Polymarket Skill）   │
              │  ① markets_search → 关联盘口           │
              │  ② clob_midpoint → 群体概率共识        │
              │  ③ clob_price_history → 概率趋势       │
              │  ④ clob_spread + clob_book → 确信度    │

              └──────────────────┬──────────────────┘
                                 │ 三角色信号数据包
                                 ▼
              ┌─────────────────────────────────────┐
  Stage 4     │  综合分析与预测（Claude AI 深度）      │
              │  输入：事件描述 + 概率信号 + 映射规则   │
              │  ① 事件影响定性分析                     │
              │  ② 预测市场信号三角色解读               │
              │  ③ 多情景构建（乐观/中性/悲观）         │
              │  ④ 价格方向概率估算                     │
              │  ⑤ 综合信号仪表盘评分                   │
              └──────────────────┬──────────────────┘

                    ┌────────────┼────────────┐
                    ▼            ▼            ▼
              方向概率判断   多情景分析    信号仪表盘
```

### 各阶段的信息守恒原则

每一层只做**信息增量**，不丢弃上一层的原始数据：

- Stage 1 输出事件原文 + 结构化标签
- Stage 2 输出映射推理过程 + 资产列表（附带 Stage 1 原始事件）
- Stage 3 输出原始盘口数据 + 信号解读（附带 Stage 1-2 的上下文）
- Stage 4 基于全量上下文生成最终预测

这确保了分析的**可追溯性**——任何预测结论都可以回溯到触发它的原始事件和数据。

## 三、事件分类与资产影响映射方法论

### 事件分类体系

| 事件类型 | 子类别 | 典型触发信号 |
|----------|--------|-------------|
| **地缘冲突** | 军事行动、制裁升级、领土争端 | 军队调动报道、联合国决议、制裁清单更新 |
| **央行政策** | 利率决议、QE/QT、前瞻指引 | FOMC 声明、ECB 决议、央行官员讲话 |
| **监管政策** | 加密监管、金融监管、反垄断 | SEC 行动、国会立法、行政令 |
| **选举与政权更迭** | 大选、弹劾、内阁更换 | 民调变化、候选人声明、投票结果 |
| **贸易与关税** | 关税调整、贸易协定、制裁 | 行政令、贸易代表声明、报复措施 |
| **经济数据冲击** | CPI 超预期、就业异常、GDP 修正 | 数据发布 vs 市场预期偏差 |
| **黑天鹅** | 疫情、自然灾害、金融机构爆雷 | 突发新闻、官方紧急声明 |

### 影响映射矩阵

映射遵循"事件 → 传导路径 → 资产影响"的因果链逻辑：

```
地缘冲突升级
  ├→ 避险需求上升 → 黄金 ↑, 美债 ↑, 日元/瑞郎 ↑
  ├→ 能源供应担忧 → 原油 ↑（产区相关时）
  ├→ 风险偏好下降 → 股指 ↓, 加密 ↓（短期）
  └→ 供应链中断   → 相关大宗 ↑, 依赖进口国货币 ↓

央行鹰派超预期
  ├→ 利率预期上行 → 美元 ↑, 债券 ↓
  ├→ 流动性收紧   → 股指 ↓（尤其成长股）, 加密 ↓
  └→ 利差扩大     → 新兴市场货币 ↓, 套利交易平仓

加密监管利好
  ├→ 合规化预期   → BTC/ETH ↑
  ├→ 机构资金入场  → 加密总市值 ↑
  └→ 监管套利消失  → 合规交易所股票 ↑, 灰色市场 ↓

贸易关税升级
  ├→ 进口成本上升 → 受影响国货币 ↓, CPI 预期 ↑
  ├→ 报复循环预期 → 全球股指 ↓, 避险资产 ↑
  └→ 供应链重组   → 受益国资产 ↑, 受损国资产 ↓
```

### 影响幅度评估框架

不是所有事件都值得分析。使用"事件影响力评分"进行筛选：

```
事件影响力 = 概率变化幅度 × 资产敏感度 × 时间紧迫性

概率变化幅度：预测市场盘口 24h 内变动的绝对值
  - > 15% → 重大（CRITICAL）
  - 5%-15% → 显著（SIGNIFICANT）
  - < 5% → 边际（MARGINAL）

资产敏感度：该事件类型对目标资产的历史影响系数
  - 直接影响（如 FOMC → 股指）→ 高敏感
  - 间接影响（如 FOMC → 加密）→ 中敏感
  - 远端影响（如 地方选举 → 大宗商品）→ 低敏感

时间紧迫性：事件分辨的时间窗口
  - 24h 内分辨 → 紧急
  - 1-7 天 → 标准
  - > 7 天 → 可跟踪
```

## 四、预测市场信号的三重角色解读框架

预测市场盘口不是单一维度的数据，而是可以从三个角色维度解读的复合信号源。

### 角色一：群体智慧信号（Crowd Wisdom）

**本质**：midpoint 价格 = 市场参与者用真金白银投票出的事件概率共识。

**使用方法**：
- 提取 `clob_midpoint` 作为事件发生概率的**锚定值**
- 与自身基于信息分析的概率判断进行对比
- 当两者偏差 > 10%，深入分析谁更可能正确

**解读要点**：
```
midpoint = 0.72 → 市场认为该事件有 72% 概率发生

关键判断维度：
① 市场信息是否充分？（看交易量和市场存续时间）
   - 日均交易量 > $50k + 存续 > 7 天 → 信息较充分
   - 日均交易量 < $10k 或新建市场 → 信号可靠性打折

② 市场参与者结构是否合理？（看 data_leaderboard）
   - 顶级交易者重仓某方向 → 信号权重提高
   - 参与者高度同质化 → 可能存在群体偏差

③ 概率是否处于"信息充分区间"？
   - 20%-80%：不确定性高，信号价值最大
   - < 10% 或 > 90%：接近共识，边际信息量低
```

### 角色二：领先指标（Leading Indicator）

**本质**：预测市场价格变动往往**早于**传统金融市场反应，因为预测市场参与者对特定事件的关注度和反应速度更高。

**使用方法**：
- 提取 `clob_price_history`，分析概率变动的**时间点和速度**
- 将概率变动时间戳与传统市场价格变动对比，寻找时间差

**典型模式**：
```
模式 A：预测市场领先反应
  T+0h  Polymarket "中美关税升级" 盘口从 35% → 60%
  T+2h  美股期货尚未明显反应
  T+4h  亚洲开盘，相关概念股开始下跌
  → 窗口期：预测市场变动后 2-4 小时内，传统市场尚未充分定价

模式 B：预测市场确认信号
  T+0h  市场传闻某央行将紧急降息
  T+1h  Polymarket 相关盘口从 10% → 45%
  → 解读：群体下注验证了传闻的可信度，增强对降息的判断置信度

模式 C：预测市场提前定价
  T-48h  Polymarket 盘口已缓慢从 20% → 55%
  T+0h   官方公布消息，盘口跳至 95%
  → 解读：信息已被部分定价，传统市场的反应幅度可能不如预期
```

**关键指标**：
- **概率变动速度**：1 小时内变动 > 10% → 强信号（可能有内部信息流出）
- **概率变动方向的持续性**：连续 3 个时间段同向变动 → 趋势确认
- **变动前的基准水平**：从低概率区（<30%）快速上升比从中间区更具信号价值

### 角色三：情绪温度计（Sentiment Gauge）

**本质**：买卖价差、委托簿深度和交易量的组合反映了市场对事件结果的**确信度和焦虑度**。

**使用方法**：
- `clob_spread`：价差越窄 → 做市商越自信 → 市场对结果有较强共识
- `clob_book`：买卖挂单不平衡 → 方向性压力 → 市场情绪偏向
- `data_trades`：大单方向 → Smart Money 的态度

**情绪状态矩阵**：

| Spread | Book 平衡度 | 交易量 | 情绪解读 |
|--------|------------|--------|---------|
| 窄 (<50bps) | 平衡 | 高 | **稳定共识**：市场对结果有较强一致预期 |
| 窄 | 不平衡 | 高 | **方向性共识**：市场强烈偏向某方向 |
| 宽 (>200bps) | 不平衡 | 低 | **不确定焦虑**：市场迷茫，做市商撤退 |
| 宽 | 平衡 | 高 | **激烈分歧**：多空对峙，波动性即将放大 |

**情绪温度转化为资产预测信号**：
```
稳定共识 + 概率 > 70% → 事件高概率发生，资产已部分定价
方向性共识（偏 YES）+ 概率快速上升 → 事件加速，资产尚未充分反应
不确定焦虑 → 市场缺乏方向，观望为主，不宜下强判断
激烈分歧 → 事件结果高度不确定，资产波动性预期上升（利好波动率策略）
```

## 五、多维分析方法论：五层递进框架

### 第一层：事件识别与定性分析

**数据源**：OpenClaw（Twitter + News）

**操作步骤**：
1. 提取事件的核心事实（谁、做了什么、什么时间）
2. 判断事件类型（参照第三章分类体系）
3. 评估事件的**新增信息量**——是全新事件还是已知事件的进展？
4. 给出事件紧急度评分（CRITICAL / SIGNIFICANT / MARGINAL）

**输出**：
```json
{
  "event_summary": "美国财政部宣布对 X 国实施全面金融制裁",
  "event_type": "地缘冲突.制裁升级",
  "novelty": "全新事件，此前无市场预期",
  "urgency": "CRITICAL",
  "timestamp": "2026-03-10T14:30:00Z"
}
```

### 第二层：概率锚定

**数据源**：Polymarket Skill

**操作步骤**：
1. 根据事件关键词搜索相关预测市场（`markets_search`）
2. 提取所有相关盘口的 midpoint 作为群体概率锚
3. 分析 price_history 判断概率变动趋势
4. 评估信号可靠性（交易量、市场存续时间、参与者结构）

**输出**：
```json
{
  "related_markets": [
    {
      "question": "Will US impose full sanctions on X by April?",
      "midpoint": 0.68,
      "24h_change": "+0.23",
      "volume_24h": "$125,000",
      "signal_reliability": "HIGH",
      "role_reading": {
        "crowd_wisdom": "市场认为制裁落地概率 68%",
        "leading_indicator": "24h 内概率跳升 23%，传统市场尚未充分反应",
        "sentiment_gauge": "spread 收窄至 30bps，方向性共识强烈偏 YES"
      }
    }
  ]
}
```

### 第三层：影响链推导

**数据源**：Claude AI（基于映射规则 + 上下文推理）

**操作步骤**：
1. 根据事件类型调用影响映射矩阵
2. 识别一阶影响（直接受影响资产）和二阶影响（联动资产）
3. 评估各传导路径的**强度**和**时延**

**输出**：
```json
{
  "impact_chains": [
    {
      "path": "全面金融制裁 → X国原油出口受阻 → 全球供应缺口",
      "affected_asset": "WTI Crude Oil",
      "direction": "BULLISH",
      "impact_strength": "HIGH",
      "time_lag": "hours",
      "order": 1
    },
    {
      "path": "制裁升级 → 地缘风险溢价 → 避险需求",
      "affected_asset": "Gold (XAU)",
      "direction": "BULLISH",
      "impact_strength": "MEDIUM",
      "time_lag": "hours",
      "order": 1
    },
    {
      "path": "制裁 → 供应链担忧 → 通胀预期上升 → 加息预期强化",
      "affected_asset": "S&P 500",
      "direction": "BEARISH",
      "impact_strength": "MEDIUM",
      "time_lag": "days",
      "order": 2
    }
  ]
}
```

### 第四层：情景构建

**操作步骤**：
1. 以预测市场概率为锚，构建三种情景
2. 每种情景赋予概率权重（三者之和 = 100%）
3. 每种情景下估算目标资产的价格区间

**框架**：
```
乐观情景（对标的物价格利好的路径）
  触发条件：[具体描述]
  概率权重：[基于预测市场信号 + 自身判断]
  资产影响：[方向 + 幅度区间 + 时间框架]

中性情景（事件影响被市场消化或对冲）
  触发条件：[具体描述]
  概率权重：[...]
  资产影响：[...]

悲观情景（对标的物价格利空的路径）
  触发条件：[具体描述]
  概率权重：[...]
  资产影响：[...]
```

**情景概率校准规则**：
- 预测市场 midpoint 作为"事件发生概率"的基准
- 事件发生 → 激活乐观或悲观情景（取决于事件对标的物的方向）
- 事件不发生 → 激活对立情景
- 中性情景覆盖"事件部分发生"或"影响被对冲"的路径

### 第五层：价格区间估算

**操作步骤**：
1. 综合三种情景的概率加权，得出**期望方向和幅度**
2. 给出**置信区间**（高/中/低置信度）
3. 明确预测的**有效时间窗口**

**输出**：
```json
{
  "target_asset": "WTI Crude Oil",
  "current_price": "$78.50",
  "direction_probability": {
    "bullish": 0.65,
    "neutral": 0.20,
    "bearish": 0.15
  },
  "expected_range": {
    "7d_high": "$84.00",
    "7d_low": "$76.00",
    "probability_weighted_target": "$81.20"
  },
```

## 六、分析引擎 System Prompt 设计架构

### 角色设定

```
你是一位专精于地缘政治经济学的资产分析师，具备以下能力：
1. 深刻理解地缘政治事件对全球金融市场的传导机制
2. 精通预测市场（Polymarket）的盘口数据解读——
   将群体下注概率转化为资产价格分析的信号输入
3. 能够构建多情景分析框架，给出概率加权的价格预测

你的分析原则：
- 预测市场盘口是群体智慧的浓缩，但不是真理——需要独立验证
- 所有预测必须明确置信度和有效时间窗口
- 区分一阶影响（直接且快速）和二阶影响（间接且滞后）
- 永远考虑"市场已定价多少"——避免对已知信息做冗余分析
```

### 分析框架指令

```
按照以下六步框架分析：

1. 事件定性评估
   - 事件的核心事实是什么？
   - 这是全新事件还是已知事件的进展？
   - 市场对此事件的预期程度如何？（参照预测市场概率变动历史）

2. 预测市场信号三维解读
   - 群体智慧维度：midpoint 概率是否合理？你的独立判断是否一致？
   - 领先指标维度：概率变动是否领先于传统市场反应？时间差有多大？
   - 情绪温度维度：spread/book/交易量反映的市场确信度和焦虑度如何？

3. 影响链推导
   - 该事件通过哪些传导路径影响目标资产？
   - 一阶影响和二阶影响分别是什么？
   - 各路径的强度和时延估计

4. 多情景构建
   - 乐观情景：触发条件、概率权重、价格影响
   - 中性情景：触发条件、概率权重、价格影响
   - 悲观情景：触发条件、概率权重、价格影响
   注意：三情景概率之和必须等于 100%

5. 综合信号评分
   - 方向强度（-1.0 到 +1.0）
   - 置信度（HIGH / MEDIUM / LOW）
   - 时间框架（IMMEDIATE / SHORT_TERM / MEDIUM_TERM）
   - 波动性预期（ELEVATED / NORMAL / COMPRESSED）

6. 风险与不确定因素
   - 哪些因素可能颠覆以上分析？
   - 预测市场信号的局限性在哪里？（流动性、参与者偏差等）
   - 建议的再评估时间点
```

## 七、系统局限与扩展方向

### 7.1 当前系统的结构性局限

**信号完整性缺口**：

| 缺失数据 | 影响 | 优先级 |
|----------|------|--------|
| 传统市场实时价格（BTC/ETH, 股指期货, 原油, 外汇） | 无法验证预测市场信号是否已被传统市场消化 | **最高** |
| 经济数据日历（FOMC, CPI, NFP 等） | 无法提前布局已知催化剂事件 | **高** |
| 跨平台预测市场价格（Kalshi, Metaculus） | 缺乏交叉验证，单一平台可能存在偏差 | **中** |
| 历史事件-资产影响数据库 | 无法量化验证影响映射矩阵的准确性 | **中** |
| VIX / 隐含波动率数据 | 缺少市场整体风险偏好的温度计 | **中** |

**分析能力局限**：

1. **概率校准问题**：预测市场在低流动性盘口上的定价可能严重偏离真实概率。当日均交易量 < $10k 时，midpoint 的信号价值大幅下降。

2. **参与者偏差**：Polymarket 用户群体以加密原生社区为主，在政治和地缘事件上可能存在系统性偏差（如对特定政治立场的过度乐观/悲观）。

3. **因果推理的局限**：影响链推导依赖 Claude AI 的知识和推理能力，在面对史无前例的事件组合时，可能出现"类比推理失效"。

4. **时间分辨率**：OpenClaw 的信息采集频率和 Polymarket API 的响应速度共同决定了系统的最快反应时间，在极端行情中可能不够及时。

### 7.2 扩展路线图

**Phase 1：数据源增强**
- 接入传统市场实时价格 API（如 Yahoo Finance / Alpha Vantage / CoinGecko）
- 建立经济事件日历模块（可使用 Trading Economics 或 Investing.com 数据）
- 实现"预测市场概率变动 vs 传统市场价格变动"的实时对比

**Phase 2：分析能力增强**
- 建立历史事件-资产影响数据库，用于回测影响映射矩阵
- 引入跨平台预测市场价格对比（Kalshi / Metaculus）作为交叉验证
- 开发情绪量化模块：将 OpenClaw 的 Twitter 信息流转化为定量情绪评分

**Phase 3：闭环反馈**
- 记录每次预测的输入、推理、输出和实际结果
- 定期回顾预测准确率，校准影响映射矩阵的系数
- 识别系统性偏差并修正

### 7.3 使用本框架的关键原则

1. **预测市场是输入，不是圣经**——midpoint 是有价值的锚，但它反映的是参与者的共识，不是真相。独立思考是不可替代的。

2. **明确置信度比精确预测更重要**——"我们对这个方向有 60% 信心"比"目标价 $85"更诚实也更有用。

3. **时间窗口是预测的一部分**——没有时间框架的价格预测是不完整的。所有输出必须包含有效期。

4. **可追溯性优先**——每个结论都应该可以追溯到触发它的事件和数据，以便事后复盘和系统改进。

5. **信号不够强时，"不做判断"是合法结论**——当预测市场信号可靠性低、影响链模糊、情景概率接近均分时，输出 PASS 比强行预测更负责。
